# 멀티 사이트 구조 전환 계획

## 개요

기존 카테고리의 **대분류(level 1) = 독립된 쇼핑몰 사이트** 역할로 격상한다.
별도의 "몰" 도메인을 신설하지 않고, **기존 대분류 카테고리가 곧 사이트 구분자**가 된다.

### 운영 방식
- **글로벌 사이트 선택기 없음** — 통합 어드민 컨셉 유지
- 기존 페이지 구조를 그대로 사용하면서, **대분류(=사이트) 필터**로 데이터를 구분
- 전체 사이트의 데이터를 한눈에 보거나, 특정 사이트로 필터링하여 관리

### 변경 전
```
대분류(level 1) → 중분류(level 2) → 소분류(level 3)
  ↑ 단순 카테고리 분류
```

### 변경 후
```
대분류(level 1) = 사이트(의류몰, 가구몰, 잡화몰 …)
  └→ 중분류(level 2) → 소분류(level 3)
       ↑ 해당 사이트 내부 카테고리 계층
```

**핵심 원칙**: 대분류 = 사이트. 새 엔티티·글로벌 상태 없이, 기존 필터 구조를 활용한다.

## Phase 1: 상품 등록/수정 — 대분류 선택 우선 UX

> 대분류 선택을 폼의 최우선 단계로 분리. 대분류를 먼저 선택해야 나머지 입력 가능

### 1-1. 상품 폼 구조 변경
- [x] `components/products/ProductForm.tsx`
  - **현재 구조**: 기본 정보 카드 안에 이미지·상품명·대분류·중분류·가격·재고가 함께 있음
  - **변경 구조**:
    ```
    [섹션 0: 대분류 선택] ← 신규. 대분류 미선택 시 아래 섹션 비활성화
    [섹션 1: 기본 정보]   ← 이미지, 상품명, 중분류, 소분류, 가격, 재고
    [섹션 2: 추가 정보]   ← 브랜드, 마진, 상태, 설명
    [섹션 3: 옵션]        ← 기존과 동일
    ```
  - 대분류 미선택 상태 → 하위 섹션 **비활성화(disabled) 또는 시각적으로 흐리게** 표시
  - 대분류 선택 시 → 하위 섹션 활성화 + 해당 대분류의 중분류/소분류만 표시 (기존 캐스케이드 로직 유지)

### 1-2. 수정 모드 처리
- [x] 수정 모드(`isEdit`)에서는 기존 상품의 대분류가 이미 선택된 상태로 진입
  - 대분류 변경 시 중분류/소분류 초기화 (기존 동작 유지)

### 1-3. 라벨 추가
- [x] `data/labels.ts` — 대분류 선택 섹션 관련 라벨 추가
  - 섹션 타이틀, 안내 문구 ("대분류를 먼저 선택해주세요" 등)
